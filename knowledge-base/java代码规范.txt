工作室 Java 代码规范（参考阿里巴巴 Java 开发手册）：

命名规范：
- 类名：大驼峰，如 UserController。
- 方法名：小驼峰，如 getUserById。
- 常量：全大写，下划线分隔，如 MAX_SIZE。
- 变量名：有意义的英文，避免拼音。

注释规范：
- 类、方法、成员变量必须写 Javadoc 注释，说明用途。
- 方法内部复杂逻辑要写行注释。
- 临时注释用 // TODO 标记。

格式规范：
- 缩进使用 4 个空格，不用 Tab。
- 左大括号不换行，右大括号换行。
- 运算符两边加空格。
- 每行不超过 120 字符。

异常处理：
- 不要捕获 Exception 或 Throwable，应捕获具体子类。
-  finally 块中释放资源。
- 业务异常使用自定义运行时异常。
